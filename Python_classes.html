Class Syntax

Reference material from CS 5001 with Dr. Molly Domino

Class: A blueprint for creating objects & defining structure & behavior

Attribute: Variables that define properties of an object (e.g., name, color, size)

Method: Functions that define behavior of an object (e.g., eat(), sleep(), run())

Object: Created from class using ClassName()

Instance: A specific occurrence of an object (e.g., luna, Bella, Max - each a unique Dog)

Self Param (1st): Assigns instance of class to object & allows access & modification to attributes & methods in the class

Constructor: __init__(self, ...) ‚ûú Always 1st Method, initializes attributes, creates new object or instance

Instance (object classname): Used to check if an object is a specific instance of a class or superclass

Class Example

class Fruit:
# Constructor
def __init__(self, name, color, taste, price, price_round=True):
self.name = name
self.color = color
self.taste = taste
self.price = price
self.price_round = price_round
# Method
def examine_fruit(self):
return f"This fruit is named {self.name}! Color: {self.color}, Taste: {self.taste}, Price: ${self.price if not self.price_round else round(self.price)}"
def check_cost_of_ten(self):
return f"Ten {self.name} will cost ${self.price * 10}"
def is_sweet(self):
return "sweet" in self.taste.lower()
# Create instances (objects)
apple = Fruit("Apple", "Red", "Sweet", 1.99)
banana = Fruit("Banana", "Yellow", "Sweet", 0.79, True)
# Use methods
print(apple.examine_fruit())
print(banana.check_cost_of_ten())

Class Testing Steps

STEP 1: Create accurate "if" for test_share or related class

STEP 2: Import class & unittest for test file or main class

STEP 3: Have a "TestClass" (for each test) that imports unittest

STEP 4: Write methods for test case attributes, methods & behavior of class

STEP 5: Use unittest assertions to check actual vs expected behavior (Remember: Self does not count as an arg since method is being called by instance)

STEP 6: Run tests, looking for "OK"
